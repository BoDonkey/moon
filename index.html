Moon

A minimal, blazing fast user interface library.

Lightweight

Weighing in at 4kb minified, Moon is one of the lightest libraries out there.

Fast

Moon uses a fast Virtual DOM, and can rerender the DOM efficiently.

Intuitive

Moon's simple API makes it easy to learn, you can get going in no time.

GETTING STARTED
Installing

First, you need to link to Moon. Link to Moon before linking to your application code. Using unpkg is recommended, as it updates as soon as npm updates.

To install with npm, run npm install moonjs

Here is the latest unpkg link:

<script src='https://unpkg.com/moonjs@0.1.0'></script>
EXAMPLE
<div id='app'>
  <h1>{{msg}}</h1>
  <input type='text' m-model='msg'/>
</div>
var app = new Moon({
  el: '#app',
  data: {
    msg: 'Hello Moon!'
  }
});

{{msg}}

INSTANCE

The instance initializes Moon. It gives it the information needed to create the virtual DOM and create all links between elements, making them reactive.

Inside this instance, you provide initial data, the element to attach a Moon element to, helper methods, custom componenets, etc.

El

Accepts - String

The element, provided in el, is the element Moon initializes on, all data stored in the instance is scoped to this element.

var app = new Moon({
  el: '#app'
});
Data

Accepts - Object

The data, provided in data, is the data that Moon compiles all mustache templates with. All data is updated in real-time.

<div id='app2'>
  {{msg}}
</div>
var app2 = new Moon({
  el: '#app2',
  data: {
    msg: 'Hello Moon!'
  }
});
{{msg}}
Methods

Accepts - Object

The methods, provided in methods, are the methods that can be used throughout an application. They are helper functions that can be used to manipulate data or the DOM.

<div id='app3'>
  {{msg}}
</div>
var app3 = new Moon({
  el: '#app3',
  data: {
    msg: 'Hello Moon!'
  },
  methods: {
    change: function() {
      app3.set('msg', 'Changed the Value!');
    }
  }
});

app3.method('change');
{{msg}}
Hooks

Accepts - Object

The hooks, provided in hooks, are methods that are called internally within Moon when specific events occur.

  • created - Called when instance has been created, everything passed to the instance has been observed.
  • mounted - Called when element has been mounted, meaning Moon has replaced the element with a Moon element.
  • updated - Called when data changes, and the element has been updated with new data.
  • destroyed - Called when instance has been destroyed, the element will not be updated anymore.
<div id='app4'>
  {{msg}}
</div>
var app4 = new Moon({
  el: '#app4',
  data: {
    msg: 'Hello Moon!'
  },
  hooks: {
    created: function() {
      console.log('App 4 Created!');
    },
    mounted: function() {
      console.log('App 4 Mounted!');
    },
    updated: function() {
      console.log('App 4 Updated!');
    },
    destroyed: function() {
      console.log('App 4 Destroyed!');
    }
  }
});
METHODS

These methods come with each new instance of Moon. They allow you to do actions associated with the instance. They can be called throughout the application, as well as inside the instance declaration itself.

They allow you to do a variety of tasks, from manipulating data to destroying the instance.

Get

Accepts - String

This returns the current value of a property you provide, in the data.

<div id='app5'>
  {{msg}}
</div>
var app5 = new Moon({
  el: '#app5',
  data: {
    msg: 'Hello Moon'
  }
});

console.log(app5.get('msg'));
Set

Accepts - String, String

This allows you to set a value in the data. This should always be used to set a value.
