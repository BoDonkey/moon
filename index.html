Moon

A minimal, blazing fast user interface library.

Lightweight

Weighing in at 4kb minified, Moon is one of the lightest libraries out there.

Fast

Moon uses a fast Virtual DOM, and can rerender the DOM efficiently.

Intuitive

Moon's simple API makes it easy to learn, you can get going in no time.

GETTING STARTED
Installing

First, you need to link to Moon. Link to Moon before linking to your application code. Using unpkg is recommended, as it updates as soon as npm updates.

To install with npm, run npm install moonjs

Here is the latest unpkg link:

<script src='https://unpkg.com/moonjs@0.1.1'></script>
EXAMPLE
<div id='app'>
  <h1>{{msg}}</h1>
  <input type='text' m-model='msg'/>
</div>
var app = new Moon({
  el: '#app',
  data: {
    msg: 'Hello Moon!'
  }
});

{{msg}}

INSTANCE

The instance initializes Moon. It gives it the information needed to create the virtual DOM and create all links between elements, making them reactive.

Inside this instance, you provide initial data, the element to attach a Moon element to, helper methods, custom componenets, etc.

El

Accepts - String

The element, provided in el, is the element Moon initializes on, all data stored in the instance is scoped to this element.

var app = new Moon({
  el: '#app'
});
Data

Accepts - Object

The data, provided in data, is the data that Moon compiles all mustache templates with. All data is updated in real-time.

<div id='app2'>
  {{msg}}
</div>
var app2 = new Moon({
  el: '#app2',
  data: {
    msg: 'Hello Moon!'
  }
});
{{msg}}

See also:

  • Get - Gets a value from data.
  • Set - Set a value of the data.
Methods

Accepts - Object

The methods, provided in methods, are the methods that can be used throughout an application. They are helper functions that can be used to manipulate data or the DOM.

<div id='app3'>
  {{msg}}
</div>
var app3 = new Moon({
  el: '#app3',
  data: {
    msg: 'Hello Moon!'
  },
  methods: {
    change: function() {
      app3.set('msg', 'Changed the Value!');
    }
  }
});

app3.method('change');
{{msg}}

See also:

  • Method - Calls a method from the listed methods.
Hooks

Accepts - Object

The hooks, provided in hooks, are methods that are called internally within Moon when specific events occur.

  • created - Called when instance has been created, everything passed to the instance has been observed.
  • mounted - Called when element has been mounted, meaning Moon has replaced the element with a Moon element.
  • updated - Called when data changes, and the element has been updated with new data.
  • destroyed - Called when instance has been destroyed, the element will not be updated anymore.
<div id='app4'>
  {{msg}}
</div>
var app4 = new Moon({
  el: '#app4',
  data: {
    msg: 'Hello Moon!'
  },
  hooks: {
    created: function() {
      console.log('App 4 Created!');
    },
    mounted: function() {
      console.log('App 4 Mounted!');
    },
    updated: function() {
      console.log('App 4 Updated!');
    },
    destroyed: function() {
      console.log('App 4 Destroyed!');
    }
  }
});
METHODS

These methods come with each new instance of Moon. They allow you to do actions associated with the instance. They can be called throughout the application, as well as inside the instance declaration itself.

They allow you to do a variety of tasks, from manipulating data to destroying the instance.

Get

Accepts - String

This returns the current value of a property you provide, in the data.

<div id='app5'>
  {{msg}}
</div>
var app5 = new Moon({
  el: '#app5',
  data: {
    msg: 'Hello Moon!'
  }
});

console.log(app5.get('msg'));
Set

Accepts - String, String

This allows you to set a value in the data. This should always be used to set a value. When setting a value, the DOM will be updated in real time.

<div id='app6'>
  {{msg}}
</div>
var app6 = new Moon({
  el: '#app6',
  data: {
    msg: 'Hello Moon!'
  }
});

app6.set('msg', 'New Message!');
{{msg}}

Go ahead, and open up the console. Try entering app6.set('msg', 'New Value Here!'), and see the sample being updated!

Method

Accepts - String

This is used to call a method that is defined in the methods of an instance.

<div id='app7'>
  {{count}}
</div>
var app7_count = 0;

var app7 = new Moon({
  el: '#app7',
  data: {
    count: app7_count
  },
  methods: {
    increment: function() {
      app7_count++;
      app7.set('count', app7_count);
    }
  }
});

app7.method('increment');
{{count}}

Go ahead, open the console, and enter app7.method('increment'). The sample will update in real time!

Destroy

This will completely destroy an instance. The DOM will not be updated anymore.

<div id='app8'>
  {{msg}}
</div>
var app8 = new Moon({
  el: '#app8',
  data: {
    msg: 'Hello Moon!'
  }
});

app8.destroy();
{{msg}}

Go ahead, open up the console and try setting a value! Since App 8 was destroyed, it will not update.

DIRECTIVES

Moon directives are utilities that you declare in your HTML. Moon simply handles the rest. Common things such as adding event listeners, conditionals, and two-way data binding are built in and can be used by adding a directive.

Directives can also be custom. So you can make reusable directives and use them in your app. All directives have a prefix of m-.

Worried about if your HTML is invalid? All directives will not be present after compiled, meaning that all HTML will be valid at runtime.

If

Accepts - String

The m-if directive checks if the property in data provided in the attribute evaluates to true. If so, the html inside will be displayed. If not, the html will not even be present.

<div id='app9'>
  <p m-if='{{condition}}'>Condition is True!</p>
</div>
var app9 = new Moon({
  el: '#app9',
  data: {
    condition: true
  }
});

Condition is True!

Go ahead, open the console and try entering app9.set('condition', false), and watch the sample update in real time!

Model

Accepts - String

The m-model directive is used for two way data binding with an input and a value from data. Everytime the user puts a value in an input with this directive, it will set the provided property in the data. As you expect, the rest of the DOM will update when this property changes as well.

<div id='app10'>
  <p>{{msg}}</p>
  <input m-model='msg'/>
</div>
var app10 = new Moon({
  el: '#app10',
  data: {
    msg: 'Hello Moon!'
  }
});

{{msg}}
